# ATMS Deployment Guide

## Prerequisites

- Docker Engine 24+
- Docker Compose v2.20+
- 2 GB RAM minimum, 4 GB recommended
- 10 GB disk for images, additional for attachments and database

## Environment Setup

1. Copy the example environment file:

```sh
cp .env.example .env
```

2. Generate an application key:

```sh
docker compose run --rm api php artisan key:generate --env
```

3. Edit `.env` and set:

| Variable | Required | Description |
|----------|----------|-------------|
| `APP_KEY` | Yes | Generated by `key:generate` |
| `APP_URL` | Yes | Public URL (e.g., `https://atms.example.com`) |
| `DB_PASSWORD` | Yes | PostgreSQL password |
| `POWER_AUTOMATE_URL` | For email | Power Automate flow endpoint |
| `POWER_AUTOMATE_TENANT_ID` | For email | Azure AD tenant |
| `POWER_AUTOMATE_CLIENT_ID` | For email | Azure AD client |
| `POWER_AUTOMATE_CLIENT_SECRET` | For email | Azure AD secret |
| `POWER_AUTOMATE_MAILBOX` | For email | Shared mailbox address |

## Build and Start

```sh
docker compose build
docker compose -f compose.yaml -f compose.production.yaml up -d
```

## Initialize Database

```sh
docker compose exec api php artisan migrate --force
docker compose exec api php artisan db:seed --force
```

## Create First Administrator

```sh
docker compose exec api php artisan atms:make-admin \
    --name="Admin" \
    --email=admin@example.com \
    --password=changeme
```

## Verification

```sh
docker compose exec api php artisan test
sh scripts/test-integration.sh
sh scripts/security-smoke.sh
```

## Production Configuration

`compose.production.yaml` adds `restart: unless-stopped` to all services. The `compose.override.yaml` (development only) is automatically merged by Docker Compose but should not be present in production.

To deploy without the override:

```sh
docker compose -f compose.yaml -f compose.production.yaml up -d
```

### Architecture

- **nginx**: Public-facing reverse proxy on port 80
- **api**: PHP-FPM application (port 9000 internal only)
- **postgres**: PostgreSQL 17 database
- **queue**: Queue worker (`queue:work --tries=3`)
- **scheduler**: Scheduler (`schedule:work`)

### Volumes

- `pgdata`: PostgreSQL data (persistent)
- `attachments`: Uploaded files (persistent)

## Scheduled Tasks

| Schedule | Task | Description |
|----------|------|-------------|
| Monday 02:00 | `erp-asset-sync` | Sync assets from ERP |
| Monday 03:00 | `erp-part-sync` | Sync parts from ERP |
| Daily 06:00 | `pm-evaluation` | Evaluate PM rules and generate work orders |

## Monitoring

- **Liveness**: `GET /api/health/live` — returns 200 if PHP-FPM is running
- **Readiness**: `GET /api/health/ready` — checks database and storage connectivity
- **Queue**: Monitor `jobs` table for failed jobs

## SharePoint Integration

ATMS is hosted as a standalone web application. SharePoint contains only a normal link to the ATMS URL. Users authenticate through ATMS itself using Sanctum SPA cookie authentication. Employee directory data is imported from SharePoint via the Power Automate employee import endpoint.

## Updating

