# Client-Facing Scope Note

The proposed system is an operational Asset Maintenance Tracking System. It will not replace the ERP. The ERP will remain the source of truth for fixed assets, parts, capitalization, disposal, depreciation, procurement, and financial records.

The maintenance system will read fixed assets and parts from ERP and maintain a local operational copy where needed. This allows the system to track asset usage readings, physical locations, maintenance requests, work orders, parts used, attachments, and maintenance history.

The first version will focus on a simple maintenance lifecycle:

**ERP-linked Asset → Maintenance Request → Maintenance Manager Approval → Work Order → Closure → Maintenance History**

Preventive Maintenance Requests will be generated by the system based on configured PM rules. Corrective Maintenance Requests will be created by users when an asset requires repair or attention.
