## Backend: make the audit-logs endpoint usable for the viewer (plan + implement)

The current `AuditLogController::index` is the **only** controller in the app that hard-codes `cursorPaginate(50)` and ignores `per_page` ‚Äî every other index endpoint (WorkOrders, MaintenanceRequests, Employees‚Ä¶) follows `min((int)$request->input('per_page', 25), 5000)`. This brings audit-logs in line with that convention and adds the two filters a real viewer needs. **Only the stub consumes this endpoint**, so the contract is free to evolve.

### Changes to `AuditLogController::index` (single file)

1. **`per_page` override (consistency fix).** Replace hard-coded `cursorPaginate(50)` with `$perPage = min((int) $request->input('per_page', 50), 500); cursorPaginate($perPage)`. Default stays 50 (no behavior change for callers that send nothing); cap is 500 (audit rows are wide with JSON blobs ‚Äî lower than the 5000 list cap deliberately). This unblocks server-side "Load more" at a sane page size.

2. **Event filter ‚Üí `LIKE` (partial match).** Change `where('event', $request->event)` ‚Üí `where('event', 'like', "%{$request->event}%")`. Lets the viewer do free-text event search server-side (e.g. `event=work_order` matches all `work_order.*` events). Existing exact-match callers still match (a full event name is a substring of itself).

3. **Date-range filters (`from` / `to`).** Add `from` and `to`, matching the **exact** `WorkOrderIndexQuery` convention: `where('created_at', '>=', $from)` / `where('created_at', '<=', $to)`. Audit viewers are almost always scoped to a time window.

4. **Keep everything else identical:** `event`, `user_id`, `subject_type`, `subject_id` exact-match filters stay; `latest('id')` default order stays; `Gate::authorize('viewAny')` stays; `actor` eager-load stays; cursor pagination stays.

### What I will NOT change
- No `AuditLogResource` (raw model serialization stays ‚Äî the frontend plan accounts for the real wire keys).
- No sort param (newest-first-by-id is correct for an audit log; `from`/`to` covers the time-window need).
- No policy/auth changes (Administrator + service only ‚Äî unchanged).
- No route change.
